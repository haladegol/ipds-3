Denial of Service (DoS / DDoS)

CRITICAL
🚨 What is this attack?

Volumetric or protocol-exhaustion attacks that systematically overwhelm network infrastructure, system bandwidth, or application layers. They operate by flooding target systems with massive quantities of traffic (e.g., UDP/TCP amplification, SYN floods, or HTTP GET/POST floods) or exploiting protocol loopholes (such as incomplete TCP three-way handshakes in SYN flood attacks). This depletes finite server resources like CPU cycles, physical memory, and connection queue tables, rendering the target entirely inaccessible to legitimate, authorized traffic.

🔬 What is this attack in HADES?

Generates high-frequency, symmetric UDP/TCP packet flows targeting a single destination IP. HADES ML classifiers instantly detect the low standard deviation in packet arrival times (IAT) and trigger the IPS to shun the source IP.

Botnet Command & Control (C2)

HIGH
🚨 What is this attack?

An infrastructure of compromised internet-facing hosts (known as bots or zombies) infected with malware that silently check in with a centralized or peer-to-peer Command and Control (C2) server. This communication is executed via highly uniform, automated connection loops (known as beacons) designed to bypass perimeter firewalls by masquerading as standard web traffic (HTTP/HTTPS on ports 80/443). Once established, these beacons allow threat actors to perform unauthorized remote execution, coordinate massive distributed attacks, exfiltrate private assets, or download next-stage payloads.

🔬 What is this attack in HADES?

Simulates cyclical heartbeat connections at highly uniform microsecond intervals. HADES Stage 2 monitors this strict temporal periodicity and locks down the infected host's network access.

🔓

Host Infiltration & Privilege Escalation

HIGH
🚨 What is this attack?

A multi-phase threat vector starting with an initial compromise (often via unpatched software exploits, remote code execution, or social engineering) to drop a reverse command shell on an internal host. Once inside, the adversary engages in stealthy lateral movement and privilege escalation. By exploiting local operating system kernel flaws, executing credential harvesting tools (like Mimikatz), or hijacking active service configurations, they escalate their permissions from an unprivileged user accounts to root or SYSTEM level, establishing permanent backdoor access.

🔬 What is this attack in HADES?

Generates atypical egress traffic and unprivileged shell packet shapes. Stage 3 neural networks flag the sudden shift in connection behavior, initiating dynamic micro-segmentation to quarantine the host.

Web Application Exploits (SQLi, XSS, RCE)

CRITICAL
🚨 What is this attack?

Exploitation vectors targeting flaws in application logic, web input sanitization, and server-side configurations. Key vectors include:
• SQL Injection (SQLi): Injecting structural database queries into input fields to bypass authentication or expose/alter databases.
• Cross-Site Scripting (XSS): Injecting malicious client-side scripts to hijack active user sessions or session cookies.
• Remote Code Execution (RCE): Exploiting deserialization or input parsing flaws to execute system commands directly on the host server.

🔬 What is this attack in HADES?

Submits flows containing exploit payload patterns inside URI headers. HADES deep packet inspection (DPI) scanners match these signatures, and the system terminates the active TCP connection.

{% endif %} {% if current_result.attack_type == 'BRUTE_FORCE' %}
🔑

Credential Brute Force Attacks

MEDIUM
🚨 What is this attack?

A systematic, high-frequency authentication cracking technique targeting administrative ingress ports (such as SSH on port 22, FTP on port 21, or RDP on port 3389). Adversaries utilize automated dictionaries, credential stuffing lists, or password-spraying scripts to try hundreds of thousands of combinations against known usernames. If successful, brute force attacks grant immediate, authorized credentials to critical administrative accounts, enabling complete remote configuration takeover and system compromise.

🔬 What is this attack in HADES?

Sends rapid login failure packets from a single source on SSH or FTP ingress channels. HADES monitors the uniform packet volume and instructs the firewall to block the source IP address immediately.
